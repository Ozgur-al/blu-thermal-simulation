  <behavior>
    - build_conformal_mesh([block_A(x=0,w=0.1), block_B(x=0.05,w=0.1)]) produces x_edges=[0.0, 0.05, 0.1, 0.15] (sorted, deduplicated)
    - ConformalMesh3D.nx == len(x_edges)-1, similarly ny, nz
    - ConformalMesh3D.dx(i) returns x_edges[i+1] - x_edges[i] (non-uniform spacing)
    - ConformalMesh3D.x_centers() returns midpoints of x intervals
    - ConformalMesh3D.node_index(ix, iy, iz) returns flat index using C-order: iz*ny*nx + iy*nx + ix
    - ConformalMesh3D.total_cells == nx * ny * nz
    - assign_voxel_materials with two overlapping blocks: last-defined block wins
    - Voxels outside all blocks get "Air Gap" material (from materials_builtin.json)
    - cells_per_interval=2 doubles the cell count in each direction by subdividing each interval
  </behavior>
  <action>
    **ConformalMesh3D** (`thermal_sim/core/conformal_mesh.py`):
    - `build_conformal_mesh(blocks, cells_per_interval=1)` collects x/y/z boundary coordinates from all blocks, sorts, deduplicates (within floating-point tolerance 1e-12), and optionally subdivides each interval.
    - Store `x_edges`, `y_edges`, `z_edges` as numpy float64 arrays.
    - Provide: `nx`, `ny`, `nz` properties, `dx(i)`, `dy(j)`, `dz(k)` methods, `x_centers()`, `y_centers()`, `z_centers()` returning 1D arrays, `node_index(ix, iy, iz)` using C-order (iz*ny*nx + iy*nx + ix), `total_cells` property.
    - IMPORTANT: Use C-order for solver flat indexing. When sending to PyVista RectilinearGrid, the visualization code will handle the Fortran-order conversion using `.ravel(order='F')`. Keep this concern separate.

    **assign_voxel_materials** (`thermal_sim/core/voxel_assignment.py`):
    - `assign_voxel_materials(mesh, blocks, air_material_name="Air Gap")` returns a `np.ndarray` of shape `(nz, ny, nx)` with dtype=object containing material name strings.
    - Initialize all cells to `air_material_name`.
    - Iterate blocks in definition order. For each block, compute boolean masks using NumPy broadcasting (cx >= block.x) & (cx < block.x + block.width) etc., then outer-product to get 3D mask, assign block.material. Last-defined wins on overlap.
    - Use `>=` for lower bound and `<` for upper bound (cell center strictly inside or on lower edge).

    **Tests** (`tests/test_conformal_mesh.py`, `tests/test_voxel_assignment.py`):
    - Test mesh edge collection with overlapping blocks
    - Test cell center computation for non-uniform spacing
    - Test node_index roundtrip with known values
    - Test cells_per_interval subdivision
    - Test material assignment with no overlap, with overlap (last-wins), and with air gaps
    - Test single-block mesh produces expected cell count
  </action>
