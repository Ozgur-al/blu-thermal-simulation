```python
class _CellEditCommand(QUndoCommand):
    def __init__(self, table, row, col, old_val, new_val, main_window):
        header = table.horizontalHeaderItem(col)
        label = header.text() if header else f"col {col}"
        super().__init__(f"Edit {label}")
        self._table = table
        self._row = row
        self._col = col
        self._old = old_val
        self._new = new_val
        self._mw = main_window

    def undo(self):
        self._mw._undoing = True
        self._table.item(self._row, self._col).setText(self._old)
        self._mw._undoing = False

    def redo(self):
        self._mw._undoing = True
        self._table.item(self._row, self._col).setText(self._new)
        self._mw._undoing = False
```

**3. Wire cellChanged on all editable tables.**

Add a guard flag in `__init__`: `self._undoing = False`

Create a method to connect a table to the undo system:

```python
def _wire_table_undo(self, table):
    # Store old values for comparison
    table._old_values = {}
    table.cellChanged.connect(lambda r, c, t=table: self._on_cell_changed(t, r, c))
    table.cellPressed.connect(lambda r, c, t=table: self._capture_old_value(t, r, c))
```

`_capture_old_value(table, row, col)` stores current text in `table._old_values[(row, col)]` before the edit.

`_on_cell_changed(table, row, col)`:
- If `self._undoing` is True, return immediately (guard against recursion)
- Get old value from `table._old_values.get((row, col), "")` and new value from `table.item(row, col).text()`
- If old == new, return (no-op)
- Push `_CellEditCommand(table, row, col, old, new, self)` onto `self._undo_stack`

Call `_wire_table_undo()` for each editable table: `materials_table`, `layers_table`, `sources_table`, `led_arrays_table`, `probes_table`.

**Alternative approach for capturing old values** (simpler and more reliable): Instead of cellPressed, use `currentCellChanged` signal or install an event filter. Actually, the simplest reliable approach is:
- Override `_add_table_row` to install an item delegate or use `itemChanged` instead of `cellChanged`. But the cleanest approach per Qt docs is:
- On each table, connect `currentCellChanged(currentRow, currentColumn, previousRow, previousColumn)` to capture the text of the cell being entered. Store in `self._pre_edit_values[(id(table), row, col)]`.
- On `cellChanged(row, col)`, compare against stored pre-edit value.
- During programmatic population (`_populate_ui_from_project`), block signals: `table.blockSignals(True)` before, `table.blockSignals(False)` after, then `self._undo_stack.clear(); self._undo_stack.setClean()`.

**8. Unsaved changes prompt — `closeEvent` and helper:**

```python
def _maybe_save_changes(self) -> bool:
    """Returns True if safe to proceed, False if user cancelled."""
    if self._undo_stack.isClean():
        return True
    reply = QMessageBox.question(
        self,
        "Unsaved Changes",
        "You have unsaved changes. Save before continuing?",
        QMessageBox.StandardButton.Save | QMessageBox.StandardButton.Discard | QMessageBox.StandardButton.Cancel,
    )
    if reply == QMessageBox.StandardButton.Save:
        self._save_project()
        return self._undo_stack.isClean()  # False if save was cancelled
    elif reply == QMessageBox.StandardButton.Discard:
        return True
    return False  # Cancel

def closeEvent(self, event):
    if self._maybe_save_changes():
        event.accept()
    else:
        event.ignore()
```

Also call `_maybe_save_changes()` at the start of `_load_project_dialog()` and `_new_project()`.
