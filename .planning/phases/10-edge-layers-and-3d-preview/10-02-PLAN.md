5. **Build assembly blocks function**: `def build_assembly_blocks(project: DisplayProject) -> list[dict]`:
   - Iterate layers bottom-to-top, tracking `z_base` (cumulative thickness).
   - For each layer, create the main block: `pv.Box(bounds=[0, W*1000, 0, H*1000, z_base_mm, z_base_mm + t_mm])` (convert to mm for display).
   - If the layer has zones (including edge_layers if available via `getattr(layer, 'edge_layers', {})`), create separate box meshes for each zone that overlay the layer block. Zone boxes have the zone's x/y/width/height converted to mm, and the same z range as the layer.
   - Return a list of dicts: `{"mesh": pv.PolyData, "color": tuple, "label": str, "z_base": float, "is_zone": bool}`.
   - Scale everything to mm for sensible 3D viewport scaling.

6. **Assembly3DWidget class**:
   ```python
   class Assembly3DWidget(QWidget):
       def __init__(self, parent=None):
           super().__init__(parent)
           layout = QVBoxLayout(self)
           layout.setContentsMargins(2, 2, 2, 2)

           # PyVista plotter
           self._plotter = QtInteractor(self)
           layout.addWidget(self._plotter.interactor)  # or self._plotter depending on API

           # Explode slider row
           slider_row = QHBoxLayout()
           slider_row.addWidget(QLabel("Explode:"))
           self._explode_slider = QSlider(Qt.Orientation.Horizontal)
           self._explode_slider.setRange(0, 100)
           self._explode_slider.setValue(0)
           self._explode_slider.valueChanged.connect(self._on_explode)
           slider_row.addWidget(self._explode_slider)
           layout.addLayout(slider_row)

           self._actors: list[tuple] = []  # (actor, base_z_mm, label)
           self._blocks: list[dict] = []
   ```

   **Methods:**
   - `update_assembly(self, project: DisplayProject) -> None`: Calls `build_assembly_blocks(project)`, clears plotter, adds each block as a mesh with its color. Stores actors with their base z-positions. Adds layer name labels using `self._plotter.add_point_labels()` at the center of each layer block. Resets camera. Calls `self._plotter.render()`. **Do NOT call `self._plotter.show()`** — QtInteractor handles its own render loop.
   - `_on_explode(self, value: int) -> None`: `explode_factor = value / 100.0 * max_explode_gap_mm`. For each cached actor, compute new z-position = `base_z + layer_index * explode_factor * gap`. Use `actor.SetPosition(0, 0, z_offset)` to translate. Call `self._plotter.render()`. **Do NOT recreate meshes** — just translate existing actors.
   - `closeEvent(self, event)`: Call `self._plotter.close()` to release VTK resources. Call `super().closeEvent(event)`.

       """update_assembly() populates the plotter with actors for each layer."""

   - After architecture template is applied
   - After zone table changes

   The `_refresh_3d_preview()` method:
   ```python
   def _refresh_3d_preview(self) -> None:
       if self._3d_dock is None or not self._3d_dock.isVisible():
           return
       try:
           project = self._collect_project()
           self._3d_widget.update_assembly(project)
       except Exception as e:
           logger.debug("3D preview refresh failed: %s", e)
   ```

   Use a debounce or guard to avoid excessive rebuilds during rapid edits. A simple approach: only refresh on `layers_table.itemSelectionChanged` + a 500ms QTimer singleShot. Or just refresh at key moments (populate, add/remove layer, template apply) rather than every cell edit.
