**Step 6: Implement _on_architecture_changed(arch: str)**

```python
def _on_architecture_changed(self, arch: str) -> None:
    index = {"Custom": 0, "DLED": 1, "ELED": 2}.get(arch, 0)
    self._led_arrays_stack.setCurrentIndex(index)
    if arch == "Custom":
        return  # No template — keep existing data

    # Read current panel dimensions from spinboxes (convert mm to m)
    w = self.width_spin.value() / 1000.0
    h = self.height_spin.value() / 1000.0

    from thermal_sim.models.stack_templates import dled_template, eled_template
    if arch == "DLED":
        template_data = dled_template(w, h)
    else:
        edge_cfg = self._eled_edge_config.currentText().lower().replace("/", "_")
        template_data = eled_template(w, h, edge_config=edge_cfg)

    self._apply_template(template_data)
```

**Step 7: Implement _apply_template(template_data: dict)**

This method populates the entire UI from template data, similar to _populate_ui_from_project but takes a template dict instead of a DisplayProject:

1. Build a temporary DisplayProject from template_data + current spinbox values (name, width, height, mesh, transient, initial_temp from current spinboxes; layers, materials, heat_sources=[], led_arrays, boundaries, probes=[] from template).
2. Block signals on all editable tables.
3. Call _populate_ui_from_project(temp_project) — this handles layers, materials, boundaries, and the custom led_arrays_table (for backward compat even though DLED/ELED panels are active).
4. Also populate the active arch panel's spinboxes from template_data["led_arrays"][0] if present:
   - For DLED: set _dled_count_x, _dled_count_y, _dled_pitch_x/y (in mm), _dled_power, offset spinboxes, zone config, LED footprint
   - For ELED: set _eled_edge_config, _eled_count, _eled_pitch (in mm), _eled_edge_offset (in mm), LED footprint
5. Unblock signals.
6. Clear the undo stack: self._undo_stack.clear() — per locked decision (silent replacement, matching load-project behavior).
7. Clear validation errors and re-validate.

Wrap ALL spinbox sets in blockSignals(True/False) to prevent signal cascade (Pitfall 3 from RESEARCH.md).

**Step 8: Update _build_project_from_ui()**

Branch on self.arch_combo.currentText():
- "Custom": Use existing TableDataParser.build_project_from_tables (unchanged)
- "DLED" or "ELED": Build LEDArray directly from panel spinboxes via new _build_led_arrays_from_arch_panel() method, then construct DisplayProject manually using the layers/materials/boundaries from tables + LED arrays from spinboxes.

New method _build_led_arrays_from_arch_panel() -> list[LEDArray]:
- If DLED: Read all _dled_* spinboxes, build LEDArray(mode="grid", ...), read zone_powers from _dled_zone_table
- If ELED: Read all _eled_* spinboxes, build LEDArray(mode="edge", ...), set panel_width/height from width_spin/height_spin
