<context>
The core insight is that ELED lateral structure is really a layer stacking problem, not a zone painting problem. The user wants to define the cross-section the same way they define the z-stack: add rows with material + thickness, and they stack automatically. For ELED this would be:

X-stack: Frame(3mm) | Air(1mm) | PCB(5mm) | Air(1mm) | LGP bulk | Air(1mm) | PCB(5mm) | Air(1mm) | Frame(3mm)

This needs:
1. A new "lateral layers" or "edge layers" concept in the data model
2. A way to define these per-edge (bottom, top, left, right)
3. Translation to MaterialZone rectangles happens internally, not in the UI
4. Possibly a 3D structure preview so user can verify the assembly

The existing zone editor (Plan 02) is still useful for custom one-off zones, but the ELED preset should work at a higher abstraction level.
