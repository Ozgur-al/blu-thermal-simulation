5. **Create `_update_node_count_label()` method**:
   ```python
   def _update_node_count_label(self) -> None:
       total_nz = 0
       for row in range(self.layers_table.rowCount()):
           nz_spin = self.layers_table.cellWidget(row, 4)
           total_nz += nz_spin.value() if nz_spin else 1
       nx = self.nx_spin.value()
       ny = self.ny_spin.value()
       count = total_nz * nx * ny
       if count > 300_000:
           self._node_count_label.setText(f"Nodes: {count:,} ⚠")
           self._node_count_label.setStyleSheet("color: #ff6b6b;")
       else:
           self._node_count_label.setText(f"Nodes: {count:,}")
           self._node_count_label.setStyleSheet("")
   ```

7. **Add 300k warning before solve**: In the run method (where `self._sim_controller.start(...)` is called), before starting the simulation, compute the node count. If > 300,000, show a QMessageBox.warning with "The model has {count:,} nodes (>300k). Large models may take significantly longer to solve." with "Proceed" and "Cancel" buttons. If user cancels, return without solving.
