    **Core functions:**

    `_build_frame_tray(panel_w, panel_d, back_h, inner_h, wall_t, material, z_base) -> list[AssemblyBlock]`:
    Decompose metal frame into 5 non-overlapping blocks per RESEARCH Pattern 4:
    - Back plate: full footprint at z_base, height=back_h
    - Left wall: x=0, y=0, z=z_base+back_h, w=wall_t, d=panel_d, h=inner_h (full depth)
    - Right wall: x=panel_w-wall_t, same pattern
    - Front wall: x=wall_t, y=0, w=panel_w-2*wall_t, d=wall_t (inset to avoid corner overlap)
    - Back wall: x=wall_t, y=panel_d-wall_t, same width pattern

    `_build_optical_films(films, inner_w, inner_d, wall_t, z_base) -> tuple[list[AssemblyBlock], float]`:
    Stack optical films bottom-to-top at z_base, each spanning inner dimensions (wall_t offset). Return blocks and z_cursor after last film.

    `_build_boundary_groups(params, ambient_c) -> list[BoundaryGroup]`:
    Collect per-face BC tuples, deduplicate faces with identical (h, radiation) into single BoundaryGroup objects.

    `_collect_materials(block_list) -> dict[str, Material]`:
    Scan all blocks, look up each material name in the builtin library. For "LED Package" (not in builtins), create it inline: k=20, density=3000, Cp=800, e=0.5.

    `generate_eled(params: EledParams) -> VoxelProject`:
    Build z-stack bottom-to-top:
    1. Back cover at z=0
    2. Frame tray (back plate + walls) at z=back_cover_h
    3. Reflector at z=back_cover_h + frame_back_h, inner dims (wall_t inset)
    4. LGP at z=reflector_top, inner dims
    5. For each active LED edge: PCB strip adhered to frame inner wall at LGP z-level; LEDs on PCB inner face with uniform pitch
    6. Optical films stacked above LGP top
    7. Panel at top
    8. Air gap blocks NOT needed — voxel solver fills empty cells with air automatically. The CONTEXT says "air must be represented as explicit air blocks" but per Pitfall 4 in RESEARCH, inner components fill the inner volume and remaining space defaults to air. Only add explicit air blocks if there are visible large gaps users would want to see in 3D view. Use discretion: skip explicit air blocks if inner stack fills the inner_w x inner_d footprint, add them only for the ELED side regions between frame walls and LGP/reflector where PCB strips do not fill the gap.
    9. Probes: add 2 default probes (LGP Centre, Panel Centre)
    10. BoundaryGroups via `_build_boundary_groups`
    11. Mesh config from params

    `generate_dled(params: DledParams) -> VoxelProject`:
    Build z-stack bottom-to-top:
    1. Back cover at z=0
    2. Frame tray at z=back_cover_h
    3. Reflector at z=back_cover_h + frame_back_h, inner dims
    4. PCB at z=reflector_top, inner dims
    5. LED grid on PCB top face: LED-R{r}-C{c} at x=offset_x + c*pitch_x - led_w/2, y=offset_y + r*pitch_y - led_d/2, z=pcb_top
    6. Air cavity block (inner dims, z=pcb_top+led_h to diffuser bottom — BUT LEDs protrude into it, so air cavity z starts at pcb_top, h=air_cavity_h, LEDs overlap it with last-defined-wins)
       Actually simpler: air cavity from pcb_top to diffuser_bottom, LEDs placed after cavity so they override. Or: skip explicit air block (solver fills with air by default). Use the default-air approach to keep it simple.
    7. Diffuser at z after PCB + air_cavity_h
    8. Optical films above diffuser
    9. Panel at top
    10. Probes, BCs, mesh same pattern

    **LED naming conventions (Claude's Discretion):**
    - ELED: "LED-L-{i}", "LED-R-{i}", "LED-T-{i}", "LED-B-{i}" for left/right/top/bottom edges
    - DLED: "LED-R{r}C{c}" (row, col, 0-indexed)
